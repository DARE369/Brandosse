## Phase 1 — `implementation-researcher`

**2026-06-23 — implementation-researcher** — Decision: Recommend keeping/refactoring the existing custom `MonthGrid.jsx`/`WeekGrid.jsx` into the new shared `src/calendar/components/CalendarGrid.jsx`, rather than adopting any third-party calendar-grid library (`react-big-calendar`, FullCalendar, etc.).
Reasoning: confirmed via direct read of `package.json:27-67` that no such library is currently a dependency — this would be a net-new addition, not a swap of something already partially in place. Weighed against four factors: (1) the existing grid already implements `CALENDAR_SPEC.md` §3's idiosyncratic "3 cards + overflow" density rule and would need bespoke cell-rendering work for the platform-icon-stack/grouping requirements (§2.2/§4) regardless of the underlying engine, eroding most of a library's time-savings; (2) Master Brief §0 rule 5 forbids any new component library / new design system surface, and calendar libraries ship their own theming model that would need wholesale overriding; (3) `@dnd-kit` is already installed, already configured with a touch-tuned sensor, and already proven in this exact codebase (`WeekGrid.jsx`) — adopting a calendar library would mean either forfeiting that investment or running two parallel DnD systems; (4) no date-manipulation library exists in this codebase today (confirmed absent from `package.json`) and most calendar libraries pull one in as a peer dependency, which would reverse a deliberate existing simplicity (`timezone.js`'s Intl-only approach).
If wrong (e.g., if Phase 2 mockup work discovers the custom grid can't reasonably support keyboard-grid navigation/ARIA roving-tabindex patterns without disproportionate effort): the fallback would be evaluating a *single* additional dependency narrowly for the accessibility gap (e.g., a roving-tabindex utility, not a full calendar library) rather than replacing the whole rendering engine — flagging this as the most plausible way this recommendation could be partially revisited, not reversed outright.
