# Personal Page Concept: Calendar

Updated: 2026-05-18

## Purpose

I am thinking about Calendar as the user's personal scheduling board.

It should help the user plan when content goes out and understand what is already scheduled, published, failed, or still waiting.

## What It Should Help With

Calendar should help the user:

- View scheduled posts.
- Reschedule posts.
- See published and failed posts.
- Find drafts that need placement.
- Use posting-time suggestions.
- Accept useful open slots.
- Schedule from library content.
- Start new content for a specific date.

## Main User Questions

Calendar should answer:

- What is going out next?
- Where are the gaps in my schedule?
- Which drafts can I schedule?
- What time should I post?
- Did anything fail?

## Ideal Flow

The user opens Calendar, sees their current posting plan, and chooses whether to schedule, reschedule, publish, or create new content for an empty slot.

## Success Outcome

Calendar succeeds if the user can keep a consistent posting rhythm without manually tracking everything elsewhere.
