# Personal Page Concept: Dashboard

Updated: 2026-05-18

## Purpose

I am thinking about the Dashboard as the user's personal command center.

It should not only show numbers. It should help the user understand the state of their content workflow and decide what to do next.

## What It Should Help With

The Dashboard should help the user see:

- Whether their workspace is set up.
- Whether a social account is connected.
- Recent generated content.
- Scheduled posts.
- Published posts.
- Draft activity.
- Account health.
- Next recommended actions.

## Main User Questions

The Dashboard should answer:

- What have I created recently?
- Is anything scheduled?
- Have I published anything yet?
- Is my account connection healthy?
- What should I do next?

## Ideal Flow

The user lands on the Dashboard after login.

If they are new, the page should guide them through setup:

1. Create account.
2. Connect a social account.
3. Generate first content.

If they are returning, the page should show current momentum and shortcuts to continue work.

## Possible Actions

The user should be able to:

- Start generating content.
- Open the calendar.
- Open a recent generation.
- Fix account connection issues.
- Continue setup.
- Move toward analytics when they need more detail.

## Success Outcome

The Dashboard succeeds if the user quickly understands their workspace status and knows the next action to take.
