# Personal Page Concept: Analytics

Updated: 2026-05-18

## Purpose

I am thinking about Analytics as the place where a personal user understands content momentum and publishing health.

It should stay simple. The goal is not to overwhelm the user with enterprise reporting, but to show whether their content workflow is improving.

## What It Should Help With

Analytics should help the user see:

- Total generated content.
- Drafts.
- Scheduled posts.
- Published posts.
- Failed posts.
- Platform activity.
- Connected account health.
- External engagement, when available.
- Suggested next actions.

## Main User Questions

Analytics should answer:

- Am I creating consistently?
- Am I publishing enough of what I create?
- Which platforms am I using most?
- Are my connected accounts healthy?
- What should I improve next?

## Ideal Flow

The user opens Analytics to understand performance and workflow health, then returns to Generate, Calendar, Library, or Settings to take action.

## Success Outcome

Analytics succeeds if it helps the user make better content and publishing decisions without needing a separate reporting tool.
