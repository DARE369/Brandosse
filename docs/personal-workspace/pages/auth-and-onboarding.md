# Personal Page Concept: Auth and Onboarding

Updated: 2026-05-18

## Purpose

I am thinking about auth and onboarding as the first path into the personal workspace.

The goal is to get the user from account creation to first useful action as quickly as possible.

## Pages and Flows

This area can include:

- Login.
- Register.
- Forgot password.
- Reset password.
- Auth callback.
- Complete signup.
- Select context.

## What It Should Help With

Auth and onboarding should help the user:

- Create an account.
- Log in.
- Recover access.
- Complete missing profile setup.
- Choose the right workspace when they belong to more than one context.
- Land in the personal workspace when appropriate.

## Main User Questions

Auth and onboarding should answer:

- How do I get in?
- What setup is still missing?
- Am I going into my personal workspace or an organization workspace?
- What should I do first?

## Success Outcome

Auth and onboarding succeeds if the user reaches the correct workspace with minimal confusion and starts the first meaningful action quickly.
